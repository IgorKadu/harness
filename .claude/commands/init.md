---
description: Prepara o OS para um projeto (novo ou existente).
---
1. Rode `os_doctor` para validar a integridade do OS.
2. Projeto NOVO: dialogue p/ entender o objetivo; vá montando a estrutura incrementalmente.
   As decisoes que forem se firmando vao para ADR + `os_remember decisions`.
3. Projeto EXISTENTE: escaneie o estado atual e alinhe o `state-of-world.md` ao que ja existe
   e ao que sera feito daqui pra frente.
4. Nada de pre-carregar tudo: o `retrieval-index.json` cresce conforme o projeto cresce.
